# LFS storage audit — herrrickshaw account, 2026-07-22

Run with `scripts/audit_lfs.py` (filtered clones, `--filter=blob:limit=2048`,
full history, no checkout — inventories every LFS pointer ever pushed without
downloading a single LFS object, which is what makes this runnable while the
budget is exhausted). 87 repos scanned. Raw data: `lfs_inventory.csv`
(1,370 pointer records), `repo_summary.csv`.

## Priority order (impact ÷ risk)

1. **Rescue `global-market-data`** (10.5y point-in-time OHLCV, partly irreplaceable —
   delisted names). Needs the LFS budget raised for one cycle OR the local
   `~/repos/global-market-data/cache_seed` copy verified complete, then convert to
   gzipped parquet as regular objects. Do this before touching anything else.
2. **Delete archive twins** after verifying their source repos (−3.67 GB, zero data
   loss — they were exact-copy insurance for the 2026-07-17 public flip).
3. **History-rewrite `global-market-research-platform` + `working-files`**
   (−2.7 GB dead weight).
4. **Collapse repo splits** (−1.8 GB): one canonical home per dataset.
5. **De-mirror formats + compress at-HEAD data**, repo by repo, per `PLAYBOOK.md`.

Steps 2–5 need no LFS downloads for repos whose data is local or re-collectable;
step 1 is the only one that may require briefly paying for a data pack.
